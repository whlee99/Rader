# Srader 프로젝트 현황 요약
1. 프로젝트 개요
프로젝트 명: Srader - ToF 센서 융합 기반 무대 조명 안전 모니터링 시스템
목표: 무대 조명 장치의 설치 및 운영 시 발생할 수 있는 기울어짐과 하부 장애물 충돌 위험을 실시간으로 감지하여 장비 파손 및 안전사고를 예방하는 것.
핵심 기능: 센서 데이터를 Raspberry Pi 4에서 수집/분석하고, 그 결과를 로컬 LCD 화면의 GUI 대시보드에 직관적으로 시각화하여 현장 작업자가 즉시 상황을 인지하고 대응할 수 있도록 지원.
2. 시스템 구성 (하드웨어 구조)
메인 프로세서: Raspberry Pi 4 Model B
기울기 감지 (S1):
센서: Benewake TFmini Plus 2개
배치: 조명 장치 양 끝단에 각각 설치.
원리: 두 센서가 측정한 거리 값의 차이를 이용해 장치의 기울기 각도를 계산.
장애물 감지 (S2):
센서: STMicroelectronics VL53L5CX 5개
배치: 조명 장치 하부에 일렬로 5개를 나란히 설치하여 넓은 감지 범위 확보.
원리: 각 센서의 중앙 8개 픽셀(ROI) 데이터를 주로 사용하여, 장비 바로 아래의 장애물 분포와 거리를 정밀하게 감지.
디스플레이: Raspberry Pi 4에 연결된 LCD 터치스크린.
3. 개발 환경 (소프트웨어)
운영체제: Raspberry Pi OS
개발 언어: Python 3.x
GUI 프레임워크: PySide6
주요 라이브러리:
PySide6.QtWidgets: 애플리케이션의 창, 레이아웃, 버튼 등 UI 요소 구성.
PySide6.QtGui: QPainter를 이용한 커스텀 위젯(수평계, 장애물 뷰) 드로잉.
PySide6.QtCore: 시그널/슬롯 메커니즘 및 데이터 업데이트를 위한 QTimer 활용.
4. GUI 특징 및 기능
현재 완성된 GUI 프로토타입은 다음과 같은 특징을 가집니다.

통합 대시보드: 다크 테마 기반의 수직 레이아웃으로, 기울기 상태, 하부 장애물, 전체 시스템 상태를 한 화면에서 명확하게 파악할 수 있도록 설계.

기울기 상태창 (TiltIndicatorWidget):

시각화: '디지털 수평계' 형태로, 기울기에 따라 색상이 변하는 버블(Bubble)이 움직여 현재 상태를 직관적으로 표현.
상태 표시: 정상(녹색), 주의(노란색), 위험(빨간색) 3단계로 상태를 구분.
정보 제공: 현재 기울기 각도(°)와 S1 센서들의 원시 거리(mm) 값을 함께 표시.
하부 장애물 감지창 (ObstacleColumnWidget):

'정면 뷰' 시점: 5개의 S2 센서 데이터를 작업자가 장비를 바라보는 시점에서 보여주어 장애물 위치를 즉시 이해할 수 있도록 함.
듀얼 뷰 모드: 사용자가 버튼으로 두 가지 시각화 방식을 선택 가능.
Bar View: 각 센서의 8개 거리 데이터를 막대그래프로 표현. 장애물의 위치와 거리를 개별적으로 파악하기 용이.
Line View: 막대그래프의 끝점을 선으로 연결한 히스토그램 형태로 표현. 장애물의 전체적인 윤곽과 형태(예: 사람 어깨)를 파악하는 데 유리.
실시간 상태 표시줄:

화면 하단에 'SYSTEM OK', 'TILT WARNING', 'OBSTACLE DETECTED' 등 시스템의 최종적인 상태를 크고 명확한 색상과 텍스트로 표시하여 즉각적인 경보 역할 수행.
현실적인 데이터 시뮬레이션:

단순 랜덤 값이 아닌, 이전 상태를 기억하여 점진적으로 변화하는 데이터를 생성.
'사람'과 같은 특정 형태의 장애물이 나타나는 시나리오(가우시안 분포 활용)를 포함하여, 실제와 유사한 환경에서 GUI의 반응성을 테스트할 수 있음.
